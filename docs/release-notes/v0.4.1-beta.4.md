**v0.4.1-beta.4** (2026-07-10) — fix macOS launch crash and Windows CI failure.

**macOS launch crash**

Beta.3 still crashed on macOS first launch with:

```
Error: Cannot find module 'better-sqlite3'
```

The root cause was not missing files in `app.asar`, but a top-level
static import of `better-sqlite3` in
`electron/services/browser/cookie-importer.ts`. That module is part
of a Windows-only cookie-import feature, yet it was pulled into the
main-process bundle on every platform, causing `require('better-sqlite3')`
at app startup on macOS.

- `fix(build): avoid top-level better-sqlite3 require on macOS` —
  moved `better-sqlite3` import inside `readBrowserCookies` as a
dynamic import, so it is only loaded when the cookie importer is
actually invoked on Windows.

**Windows CI failure**

Beta.1–beta.3 failed the Windows job because the "Verify agent-bundle"
step used bash syntax (`if [ ! -f ... ]`) on the default PowerShell
runner.

- `fix(ci): fix Windows verify agent-bundle bash syntax` — added
  `shell: bash` to the step so the script runs on GitHub-hosted
  Windows runners.

**Result**

macOS `.dmg` now launches without the better-sqlite3 require error.
Windows build reaches packaging instead of failing during workspace
build verification.
