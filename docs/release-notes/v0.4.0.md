**v0.4.0** (2026-07-09) — first stable release on the `0.4.x` line,
graduating from the `0.2.0-beta.*` / `0.3.0` track.

This release consolidates two major bodies of work landed since
`v0.3.0`: (1) a **pre-release stability audit** that hardened the
agent across six domains (LLM clients, agent core, tools, worker/IPC,
database, research mode), and (2) the **canvas smart-layout v2
stack** in `@duya/conductor` (RBush spatial index, alignment-first
snap, passive push-aside, viewport-aware packing, auto layout
tools). Together they close the gap between the beta feature set and
a shippable desktop agent.

**Headline**

The headline of this stable release is **agent stability at scale**.
A six-domain audit dispatched parallel investigations into every
layer of the HTTP+SSE stack and fixed 50+ latent defects: abort
signals now actually reach LLM HTTP requests, streaming usage no
longer loses input/cache tokens, compaction preserves
tool_use/tool_result pairing, the incremental save queue no longer
drops messages on reject chaining, the worker process pool recovers
from ERROR before respawning, and SQLite now uses WAL + soft-delete
across all platforms. The canvas side lands a production-grade
spatial index and auto-layout tooling so large boards stay
interactive.

**Highlights**

- **Agent stability audit (50+ fixes across 6 domains)**
  - LLM: AbortSignal propagation to HTTP requests; 120s idle stream
    timeout; message_delta usage tracking preserves input/cache
    tokens from message_start; HTTP 409 reclassification; no
    duplicate retries after yielded content; SSRF guard for base
    URLs; probe failure no longer poisons model-capability cache.
  - Agent core: `adjustSliceBoundary` shared helper preserves
    tool_use/tool_result pairing across all compaction strategies;
    120s timeout + child abort controller for summarizers;
    `onMessagesCompacted` callback for persistence; incremental
    save queue reject chaining + `markFlushed` resolve;
    `drainQueuedChatStart` atomic check.
  - Tools: WorkerPool workerId lookup + backgrounded worker release;
    ReadTool LRU cache (1000 entries) with mtime+size fingerprint
    dedup; EditTool atomic rename + CRLF normalization; WriteTool
    path traversal check; BashTool sensitive-key redaction + /tmp
    fallback; GrepTool execFile replacement + walkDir maxDepth;
    SubagentTool immutable token_usage + lastPersistedIndex;
    ToolRetryExecutor skips non-idempotent tools.
  - Worker/IPC: SSE close timeout + `req.on('close')` interrupt;
    state-transition locks and ERROR->IDLE before spawning;
    intentional kill flag; multiLineBuffer scope fix; PortClient
    real reconnect; `process.send` try/catch; worker-protocol
    stdout backpressure + newline escape.
  - Database: WAL mode on all platforms; generation inside
    transaction; soft delete (`status='superseded'`) replaces
    DELETE; `INSERT OR IGNORE` for message dedup; conductor
    field-level merge with `safeParseJson`; configurable
    DatabaseBridge timeout.
  - Research mode: signal parameter on all 14 callback sites;
    consecutive error counter + `throwIfCancelled` guards;
    `maxIterations` fix; `research_warning` event handling.

- **Canvas smart-layout v2 (`@duya/conductor`)**
  - RBush spatial index wrapper for O(log n) hit-testing on large
    boards.
  - Alignment-first snap policy (edge/center) replaces grid snap;
    passive push-aside collision during drag.
  - Four-corner square resize handles with `resizeMode` support.
  - Geometric hit-test with fuzzy padding.
  - Viewport-aware packing (`viewportAwarePack`) + zoom-to-fit;
    sidebar resize triggers auto zoom-to-fit.
  - Flow layout + guillotine bin-packing algorithms.
  - `canvas_auto_layout` and `canvas_apply_layout` agent tools;
    `canvas.auto_layout` RPC action.
  - `ElementMetadata` extended with layout hints; canvas capture
    hook + error boundary.

- **Mode architecture (Plan 224)**
  - Conductor migrated to `ModeModifier` declarative registry;
    widget design specs added to knowledge base.
  - Unified mode state across UI (`activeModes: Set<ModeModifierId>`)
    with sidebar, file preview, and chat enhancements.

- **Project references panel**
  - New `references` IPC handlers + preload API for curating
    per-project reference files.
  - `.duya` / `.agents` / `.claude` dot-directories now visible in
    the file tree.
  - Agent prompt gains a `references` dynamic section;
    `AddReferencesModal` + `ReferencesPanel` UI components.

- **Interagent MessageSession**
  - Resolved deadlock and event-delivery failures in
    inter-agent `MessageSession` tool (router listener + state
    transition adjustments).

- **DB self-repair and worker ready fallback**
  - DB self-repair on boot; worker ready IPC fallback path;
    message truncation guard.

- **UI polish**
  - Unified panel layout (PanelHeader, PanelZone,
    PanelFileTreeSplit, TerminalPanel registry).
  - New `MessageSessionToolRow` + refreshed `TaskToolRow`,
    `VisionToolRow`, `ToolActionRow`.
  - Sidebar button alignment, skill popover styling, thread list
    item updates.
  - i18n additions (en + zh).

**Breaking changes**

None. The `ModeModifier` migration is additive; existing
`AgentProfile` and `PermissionMode` surfaces are unchanged.
