# v0.1.4

> 2026-06-09 · first non-beta of the 0.1 line. 41 commits since
> `v0.1.3-beta.5`, covering research-mode, the new `@duya/cli`
> package, multi-modal `ReadTool`, Windows reliability fixes, and
> permission / bypass-mode correctness.

## Highlights

- **Research mode v2** — full orchestrator rewrite with dynamic step
  ordering, `StopDecision` gate, source evaluator, dedup via LLM hash,
  and an activity event stream the UI can subscribe to. The new
  `ResearchActivityPanel` ships Timeline / Plan / Sources / Report
  tabs and renders Plan / Running / Completed cards directly in the
  message column.
- **`@duya/cli` package** — desktop control plane (status, plugin,
  session, skill, mcp, provider, doctor, channel, cron, message,
  install, gateway, config, update, backup, security) is now an
  independent Commander-based package. `duya_config` is folded into
  `duya config ...` / `duya mcp add|remove|assign`.
- **Multi-modal `ReadTool` (plan 106)** — in-process Node file parser
  for PDF / DOCX / PPTX / images, with magic-byte detection, ENOENT
  path suggestions, paragraph-aware truncation, malware reminder
  injection, env-tunable limits, and a shared parse cache with the
  Electron `DocumentParserService`. 15 new test files / ~150 tests.
- **Windows / packaged-app reliability** — daemon `EADDRINUSE`
  self-heal, renderer crash logging, packaged CLI smoke-test fixes,
  `chat:interrupt` graceful cancellation with a 2 s SIGKILL grace
  window, shell-tool cwd resolved to the live project (asar-safe).
- **Permission & bypass-mode correctness** — bypass mode now
  short-circuits `BashTool` / `Write` / `Edit` permission checks,
  `canUseTool` fail-open declares `behavior: 'allow'`, the in-flight
  permission prompt is protected from SSE / phase races, and
  `usePermissions` dedupes replays on resubscribe.
- **Provider-usage overhaul** — `detectProvider(providerType, baseUrl)`
  resolves canonical ids (`minimax` / `minimax-cn` / `glm` / `glm-cn`),
  surfaces per-provider error reasons, and powers the new
  `ProviderQuotaView` / `ProviderQuotaCard`.
- **SelfImprover persistence** — background skill-review counters
  (`itersSinceSkill`, `toolCallsSinceSkill`, last review timestamp)
  are persisted to `~/.duya/self-improver-state.json` so the
  "every N turns" trigger survives across queries.
- **PowerShell tool** — first-class `PowerShellTool` extending
  `BashTool` with dedicated security rules (`Remove-Item`,
  `Invoke-Expression`, `Stop-Process`, …), read-only dry-run check,
  and shared shell-provider / shell-intelligence layer.
- **CLI control-plane expansion (plan 200)** — `duya update` (status /
  check / download / install), `duya backup` (plan / create / verify /
  restore dry-run), `duya security` (audit with 8 checks + auto-fix).
- **MCP inventory service** — read-only aggregate across
  ConfigManager, agentSettings, and on-disk manifests; exposed via
  `mcp:inventory:snapshot` IPC and rendered in the settings UI.
- **Capability management aggregate (plan 83b)** — single
  read-only snapshot of installed plugins and their capabilities,
  SSE-driven reload loop, cross-source inventory, and the new
  PluginDetailView tabbed sub-views.

  parent SSE so `SubAgentPanel` does not stay stuck on "正在运行…".

- `SelfImprover` persists counters across queries.
- `Memory` tool — stable entry ids (`<!--id:m-xxxxxxxx-->`) and

  files.

## Packaging & build

- `electron:build` no longer auto-runs `build:docparser`;
  `extraResources` for `build/document-parser` is opt-in (warns,
  does not fail) for `.doc` fallback support.
- `scripts/build-cli-bundle.mjs` produces the CJS bundle for
  `@duya/cli`.
- `package-lock.json` synced for `pdf-parse` + `node-ensure`
  transitive dependencies; workspace version bumped to `0.1.4`.
- **Workspace build order fix** — `@duya/agent` imports
  `@duya/cli/contract` (resolved via `package.json` `exports` to
  `packages/cli/dist/contract/index.d.ts`), so a clean checkout
  required `npm run build:cli` before any agent tsc. `typecheck:all`,
  `build:agent`, and `electron:build` now build cli first; the
  release workflow's `Install dependencies` step is followed by an
  explicit `Build workspace packages` step (cli → agent → gateway)
  to guarantee tsc and esbuild see the cli dist on every platform.
- `.gitignore`: `packages/cli/bundle/`, `_dbg-out.txt`,
  `.codex-artifacts/`.

## Upgrade notes

- `duya_config` is removed; migrate to `duya config ...` /
  `duya mcp add|remove|assign`. Audit events continue to use
  `X-Duya-Invoked-By` for provenance.
- `node-ensure` / `pdf-parse` transitive dependencies are now
  pinned in `package-lock.json`; run `npm ci` on a clean install.
- `chat:interrupt` is graceful (2 s grace); legacy callers that
  relied on immediate kill should re-test.
- Background sub-agents now flush a terminal progress event after
  `chat:done`; consumers should tolerate late events.
