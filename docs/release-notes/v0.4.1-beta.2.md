**v0.4.1-beta.2** (2026-07-10) — fix macOS CI build crash from beta.1.

Beta.1 failed to publish macOS/Windows artifacts due to two CI
bugs: (1) empty `CSC_LINK` secret made electron-builder treat the
empty string as a certificate file path, crashing with "not a
file"; (2) `verify-packaged-parity.mjs` exited non-zero on
optional `document-parser` items, failing the build after publish.

**Fixes**

- `fix(ci): unset empty CSC_LINK to prevent macos unsigned build
  crash` — unset empty signing env vars and set
  `CSC_IDENTITY_AUTO_DISCOVERY=false` so the build proceeds
  unsigned when no Apple Developer certificate is configured.
- `fix(ci): exclude optional checks from verify-packaged-parity
  exit code` — optional `document-parser` missing items no longer
  cause `process.exit(1)`. Only required-check failures fail the
  build.

**Result**

macOS build now produces an unsigned `.dmg` (launchable via
right-click → Open to bypass Gatekeeper). Linux continues to
publish `.deb` / `.AppImage` / `.rpm`. Windows build remains
blocked by a pre-existing `@duya/conductor` two-pass build issue
(see beta.1 notes).
