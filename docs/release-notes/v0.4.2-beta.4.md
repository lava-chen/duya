**v0.4.2-beta.4** (2026-07-15) — release pipeline fixes for macOS arm64 and Windows.

**Release pipeline (CI)**

- `Build and Release` step now declares `shell: bash` on all platforms.
  The unset-and-defaults script used POSIX `[ -z "$VAR" ]` and `unset`,
  which Windows PowerShell cannot parse (ParserError "Missing '(' after
  'if'"). Pinning to bash lets the same step run unchanged on
  windows-2022, macos, and ubuntu runners.
- macOS arm64 runners now pass `--arm64` to `electron-builder --mac`.
  Without the flag, electron-builder defaults to `arch=x64` on an
  arm64 host; `after-pack` then cross-compiles `better_sqlite3.node`
  from arm64 to x64 via `electron-rebuild`, and
  `verify-packaged-parity` flags the result as FATAL (binary x64 but
  host arm64). The script detects the host arch (`uname -m`) and
  passes the matching flag. Apple Silicon is now the supported macOS
  target; Intel Macs are no longer published from this CI matrix.
- `verify-packaged-parity` SUMMARY block now branches on the `failed`
  counter instead of the optional-only `missingItems.length`, so the
  printed message matches the actual exit code. Previously the script
  printed "All N checks passed" while exiting 1 when the native-module
  architecture check failed.

**Agent**

- `use vm_stat` for accurate macOS available-memory detection.
- Guard `initAgent` against null `providerConfig`.
