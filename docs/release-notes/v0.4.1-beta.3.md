**v0.4.1-beta.3** (2026-07-10) — fix packaged app crash on first launch.

Beta.2 produced a macOS `.dmg`, but launching the installed app
crashed immediately with:

```
Error: Cannot find module 'better-sqlite3'
```

The Electron main bundle marks `better-sqlite3` as external, and
`scripts/build-electron.mjs` copies the native module into
`dist/node_modules/` for production. However,
`electron-builder.yml` had a blanket `!node_modules` exclusion that
also stripped `dist/node_modules/`, so the packaged `app.asar`
contained no better-sqlite3 loader for the external require to
resolve.

**Fix**

- `fix(build): include dist/node_modules native modules in packaged
  app.asar` — explicitly include `dist/node_modules/better-sqlite3/`,
  `bindings/`, `file-uri-to-path/`, and `node-pty/` before the
  global `!node_modules` exclusion so native modules survive
  packaging.

**Result**

Packaged app (macOS `.dmg` and Windows installer) now resolves
`better-sqlite3` on launch and initializes the database correctly.

**Pre-release verification**

- `npm run electron:build` — exit 0.
- `node scripts/verify-packaged-parity.mjs` — required checks pass.
