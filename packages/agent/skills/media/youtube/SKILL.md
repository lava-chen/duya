# YouTube Video Platform

Extract content from YouTube videos using the browser tool's evaluate operation.

## When to Use

Use this skill when the user wants to:
- Extract transcripts/subtitles from a YouTube video
- Get video metadata (title, channel, views, etc.)
- Read comments from a video
- Search for videos on YouTube
- Summarize or analyze video content

## Required Tool

This skill requires the **browser** tool. Use `browser_navigate` to load pages and `browser_evaluate` to extract data.

## Quick Start

### Extract Video Transcript

```
1. browser_navigate: https://www.youtube.com
2. browser_evaluate: Get INNERTUBE_API_KEY from ytcfg
3. browser_evaluate: Call /youtubei/v1/player with ANDROID client to get caption URL
4. browser_evaluate: Fetch caption XML and parse with string-based parser (NO DOMParser)
5. Format and return transcript data
```

### Get Video Info

```
1. browser_navigate: https://www.youtube.com/watch?v={VIDEO_ID}
2. browser_evaluate: Extract metadata from ytInitialPlayerResponse
```

## Detailed Operations

### 1. Extract Transcript/Subtitles

**IMPORTANT**: YouTube Web client's caption URLs require a PoToken (proof of origin) generated by BotGuard at runtime. The Android client returns caption URLs that work without PoToken — same approach used by youtube-transcript-api (Python).

**Step 1: Navigate to YouTube homepage to establish session**
```json
{"operation": "navigate", "url": "https://www.youtube.com"}
```

**Step 2: Get INNERTUBE_API_KEY and fetch caption data via Android InnerTube API**
```json
{
  "operation": "evaluate",
  "script": "(async function(){ const cfg=window.ytcfg?.data_||{}; const apiKey=cfg.INNERTUBE_API_KEY; if(!apiKey)return {error:'INNERTUBE_API_KEY not found'}; const videoId='VIDEO_ID'; const resp=await fetch('/youtubei/v1/player?key='+apiKey+'&prettyPrint=false',{method:'POST',credentials:'include',headers:{'Content-Type':'application/json'},body:JSON.stringify({context:{client:{clientName:'ANDROID',clientVersion:'20.10.38'}},videoId})}); if(!resp.ok)return {error:'HTTP '+resp.status}; const data=await resp.json(); const renderer=data.captions?.playerCaptionsTracklistRenderer; if(!renderer?.captionTracks?.length)return {error:'No captions available'}; const tracks=renderer.captionTracks; const available=tracks.map(t=>t.languageCode+(t.kind==='asr'?' (auto)':'')); let track=null; const langPref=''; if(langPref){track=tracks.find(t=>t.languageCode===langPref)||tracks.find(t=>t.languageCode.startsWith(langPref));} if(!track){track=tracks.find(t=>t.kind!=='asr')||tracks[0];} return {captionUrl:track.baseUrl,language:track.languageCode,kind:track.kind||'manual',available}; })()"
}
```

**Step 3: Fetch and parse caption XML with string-based parser (NO DOMParser)**
```json
{
  "operation": "evaluate",
  "script": "(async function(){ const captionUrl='CAPTION_URL_FROM_STEP_2'; const resp=await fetch(captionUrl); const xml=await resp.text(); if(!xml?.length)return {error:'Empty response'}; function getAttr(tag,name){ const needle=name+'=\"'; const idx=tag.indexOf(needle); if(idx===-1)return ''; const valStart=idx+needle.length; const valEnd=tag.indexOf('\"',valStart); if(valEnd===-1)return ''; return tag.substring(valStart,valEnd); } function decodeEntities(s){ return s.replaceAll('&amp;','&').replaceAll('&lt;','<').replaceAll('&gt;','>').replaceAll('&quot;','\"').replaceAll('&#39;',\"'\"); } const isFormat3=xml.includes('<p t=\"'); const marker=isFormat3?'<p ':'<text '; const endMarker=isFormat3?'</p>':'</text>'; const results=[]; let pos=0; while(true){ const tagStart=xml.indexOf(marker,pos); if(tagStart===-1)break; let contentStart=xml.indexOf('>',tagStart); if(contentStart===-1)break; contentStart+=1; const tagEnd=xml.indexOf(endMarker,contentStart); if(tagEnd===-1)break; const attrStr=xml.substring(tagStart+marker.length,contentStart-1); const content=xml.substring(contentStart,tagEnd); let startSec,durSec; if(isFormat3){ startSec=(parseFloat(getAttr(attrStr,'t'))||0)/1000; durSec=(parseFloat(getAttr(attrStr,'d'))||0)/1000; }else{ startSec=parseFloat(getAttr(attrStr,'start'))||0; durSec=parseFloat(getAttr(attrStr,'dur'))||0; } const text=decodeEntities(content.replace(/<[^>]+>/g,'')).split('\\\\n').join(' ').trim(); if(text){ results.push({start:startSec,end:startSec+durSec,text}); } pos=tagEnd+endMarker.length; } if(results.length===0)return {error:'Parsed 0 segments'}; return results; })()"
}
```

**Caption Languages:**
- `en` / `en-US` - English (manual or auto-generated)
- `zh-Hans` - Chinese (Simplified)
- `zh-Hant` - Chinese (Traditional)
- `ja` - Japanese
- `ko` - Korean
- `es` - Spanish
- `fr` - French
- `de` - German

**Auto-generated vs Manual:**
```json
{
  "operation": "evaluate",
  "script": "(function(){ const tracks=window.ytInitialPlayerResponse?.captions?.captionTracks||[]; return {autoGenerated:tracks.filter(t=>t.kind==='asr').map(t=>t.languageCode),manual:tracks.filter(t=>!t.kind).map(t=>t.languageCode)}; })()"
}
```

**Note:** Some videos don't have captions (live streams, age-restricted, disabled by uploader).

### 2. Extract Video Metadata

Metadata is in `window.ytInitialPlayerResponse.videoDetails` and `window.ytInitialData`.

```json
{
  "operation": "evaluate",
  "script": "(function(){ const p=window.ytInitialPlayerResponse; const d=window.ytInitialData; if(!p?.videoDetails)return null; const details=p.videoDetails; const micro=p.microformat?.playerMicroformatRenderer; return {videoId:details.videoId,title:details.title,description:details.shortDescription,duration:parseInt(details.lengthSeconds),author:details.author,channelId:details.channelId,viewCount:parseInt(details.viewCount),publishDate:micro?.publishDate,uploadDate:micro?.uploadDate,category:micro?.category,isLive:details.isLiveContent,isPrivate:details.isPrivate,tags:details.keywords||[],thumbnail:details.thumbnail?.thumbnails?.pop()?.url}; })()"
}
```

**Fields:**
| Field | Description |
|-------|-------------|
| videoId | Video ID |
| title | Video title |
| description | Full description |
| duration | Duration in seconds |
| author | Channel name |
| channelId | Channel ID |
| viewCount | View count |
| publishDate | Publish date (ISO) |
| tags | Video tags |
| isLive | Is live stream |

### 3. Extract Comments

Comments require scrolling and extracting from `window.ytInitialData`.

**Step 1: Navigate and scroll to comments**
```json
{"operation": "navigate", "url": "https://www.youtube.com/watch?v=dQw4w9WgXcQ"}
```

Scroll down to load comments:
```json
{"operation": "scroll", "direction": "down", "amount": 800}
```

**Step 2: Extract comments**
```json
{
  "operation": "evaluate",
  "script": "(function(){ const data=window.ytInitialData; if(!data)return[]; const contents=data.contents?.twoColumnWatchNextResults?.results?.results?.contents; if(!contents)return[]; for(const item of contents){ const section=item.itemSectionRenderer; if(section?.sectionIdentifier==='comment-item-section'&&section.contents){ return section.contents.slice(0,20).map(c=>{ const r=c.commentThreadRenderer?.comment?.commentRenderer; if(!r)return null; return {id:r.commentId,text:r.contentText?.runs?.map(t=>t.text).join(''),author:r.authorText?.simpleText,likes:r.voteCount?.simpleText,time:r.publishedTimeText?.runs?.[0]?.text,isHearted:!!r.creatorHeart}; }).filter(Boolean); } } return []; })()"
}
```

### 4. Search Videos

**Step 1: Navigate to search**
```json
{"operation": "navigate", "url": "https://www.youtube.com/results?search_query=machine+learning"}
```

**Step 2: Extract results**
```json
{
  "operation": "evaluate",
  "script": "(function(){ const data=window.ytInitialData; if(!data)return[]; const sections=data.contents?.twoColumnSearchResultsRenderer?.primaryContents?.sectionListRenderer?.contents; if(!sections)return[]; const videos=[]; for(const section of sections){ const items=section.itemSectionRenderer?.contents; if(!items)continue; for(const item of items){ const v=item.videoRenderer; if(!v)continue; videos.push({videoId:v.videoId,title:v.title?.runs?.map(t=>t.text).join(''),description:v.detailedMetadataSnippets?.[0]?.snippetText?.runs?.map(t=>t.text).join('')||v.descriptionSnippet?.runs?.map(t=>t.text).join(''),duration:v.lengthText?.simpleText,viewCount:v.viewCountText?.simpleText,publishTime:v.publishedTimeText?.simpleText,channel:v.ownerText?.runs?.[0]?.text,channelId:v.ownerText?.runs?.[0]?.navigationEndpoint?.browseEndpoint?.browseId,thumbnails:v.thumbnail?.thumbnails}); } } return videos.slice(0,20); })()"
}
```

**Search Filters:**
Add `sp` parameter to URL:
- Sort by upload date: `sp=CAMSAhAB`
- Sort by view count: `sp=CAMSAhAC`
- Filter video only: `sp=EgIQAQ%3D%3D`

Example: `https://www.youtube.com/results?search_query=python&sp=CAMSAhAB`

## Tips

1. **Video IDs are case-sensitive** - `dQw4w9WgXcQ`
2. **Use InnerTube Android API for transcripts** - Avoids PoToken/BotGuard issues
3. **Use string-based XML parsing** - DOMParser may fail with TrustedHTML policies
4. **Handle missing data** - Not all videos have captions or comments
5. **Scroll for lazy loading** - Comments need scrolling to load
6. **Check for null** - Always check if data exists before accessing properties
7. **YouTube is a SPA** - Navigation may not reload page, use evaluate to get fresh data

## Common Issues

| Issue | Solution |
|-------|----------|
| DOMParser fails / TrustedHTML error | Use string-based XML parsing (see Step 3) |
| Fetch returns empty caption | Use InnerTube Android API instead of ytInitialPlayerResponse |
| No caption data | Video may not have captions or they're disabled |
| Empty comments | Comments may be disabled or need more scrolling |
| Data is stale | YouTube is a SPA, re-evaluate after navigation |
| Age-restricted | Some content requires login |

## Technical Notes

### Why InnerTube Android API?

YouTube's Web client caption URLs require a PoToken (proof of origin) generated by BotGuard at runtime. This token expires quickly and cannot be easily generated outside the official YouTube web app.

The Android client (`clientName: 'ANDROID'`) returns caption URLs that work without PoToken. This is the same approach used by:
- `youtube-transcript-api` (Python library)
- OpenCLI's YouTube transcript command

### Why String-based XML Parsing?

Browser environments with strict Content Security Policy (CSP) or Trusted Types may block `DOMParser` from parsing arbitrary XML strings. The string-based parser:
- Works in all browser environments
- Handles both standard `<text>` and srv3 `<p>` formats
- Has no external dependencies
